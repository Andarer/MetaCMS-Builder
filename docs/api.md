# Документация ИИ-Интеграции (Gemini AI API) — MetaCMS Builder 🧠

Платформа **MetaCMS Builder** имеет встроенную интеграцию с ИИ-моделями Google Gemini для автоматической генерации архитектурных планов, полей таблиц СУБД и Flutter-кода.

---

## 🤖 Выбираемые ИИ Модели и Назначение

Через интерфейс страницы настроек («SettingsScreen») пользователь может переключать активную модель:
1. **`gemini-3.5-flash`** (По умолчанию):
   - Высокая скорость ответа, минимальное потребление токенов.
   - Оптимальна для быстрого построения списков ТЗ, составления описания полей таблиц и генерации простых демонстрационных записей в Sandbox.
2. **`gemini-3.1-pro-preview`**:
   - Глубокие аналитические способности.
   - Предназначена для проектирования сложных взаимосвязей между базами (многие-ко-многим), генерации сложных SQL триггеров и комплексных Flutter/Dart контроллеров.

---

## 📋 Формат взаимодействия и Системные инструкции

Для обеспечения стабильного парсинга сгенерированных колонок утилита `CmsViewModel` передает ИИ системные промпты следующего формата:

```
Вы являетесь Главным Архитектором Систем MetaCMS. Пользователь запрашивает модуль СУБД/CRM/магазина.
Сгенерируйте ответ, включающий:
1. Концепт модуля разработки.
2. Массив структуры полей базы данных в виде markdown-блока ```json:
   [
     {"name": "field_name", "type": "String", "description": "Пояснение к полю"}
   ]
3. Рабочий Flutter/Dart-класс, представляющий данную сущность.
Пишите на языке: Русский.
```

### Пример сетевого payload-запроса (JSON):
```json
{
  "contents": [
    {
      "parts": [
        {"text": "Спроектируй систему отслеживания заказов службы доставки устриц"}
      ]
    }
  ],
  "systemInstruction": {
    "parts": [
      {"text": "Системный промпт..."}
    ]
  }
}
```

## 🔒 Безопасность и хранение ключей API

1. **Где регистрировать:** Ключ генерируется на платформе Google AI Studio.
2. **Как хранить:**
   - Строго запрещено жестко кодировать ключ в репозитории.
   - Зарегистрируйте ключ `GEMINI_API_KEY` в **Secrets Panel** среды Google AI Studio. Ключ встраивается в переменную `BuildConfig.GEMINI_API_KEY` на этапе компиляции Android-приложения.
