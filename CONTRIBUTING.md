# Руководство по участию в разработке (Contributing Guide) — MetaCMS Builder 🤝

Мы рады, что вы решили внести свой вклад в развитие **MetaCMS Builder**! Проект является открытым (Open Source), и мы ценим вклад каждого участника.

Пожалуйста, уделите несколько минут для ознакомления с нашими архитектурными нормами и стандартами кодирования.

---

## 🎨 Стандарты дизайна и кода (Kotlin & Jetpack Compose)

1. **Material Design 3 (M3):**
   - Используйте только централизованные темы и цвета (`MaterialTheme.colorScheme`). Избегайте жестко закодированных шестнадцатеричных (hex) цветов в Composables.
   - Минимальная интерактивная область кнопок должна составлять не менее **48.dp x 48.dp** для соблюдения стандартов доступности (Accessibility).
2. **Локализация (i18n):**
   - Все строки должны находиться в `Localization.kt` или `res/values/strings.xml`.
   - Не хардкодьте текстовые сообщения на экранах UI. Используйте вызов `viewModel.t("key")`.
3. **Безопасность (Secrets):**
   - Строго запрещено добавлять API-ключи, токены GitHub или секреты Gemini в открытый код или файл `local.properties`. Все секреты инжектируются через Gradle BuildConfig и Secrets Panel среды Google AI Studio.

## 🏗️ Процесс отправки изменений (Pull Requests)

1. **Форкните репозиторий** и создайте новую тематическую ветку от `main`:
   ```bash
   $ git checkout -b feature/amazing-feature
   ```
2. **Проверьте работоспособность кода:**
   Перед отправкой коммита убедитесь, что приложение успешно компилируется:
   ```bash
   $ ./gradlew compileDebugKotlin
   ```
3. **Опишите ваши изменения:**
   Добавьте четкую информацию в `RELEASE_NOTES.md` и обновите раздел `CHANGELOG.md` в соответствии с нашими категориями (Added, Changed, Fixed, Removed, Security).
4. **Отправьте Pull Request:**
   Убедитесь, что ваш PR нацелен на ветку `main`. Опишите решенную проблему и прикрепите скриншоты, если изменяется пользовательский интерфейс.

## 📋 Проверка качества у линтеров (Code Quality)

В проекте настроены статические инспекторы кода и компиляции. Перед коммитом мы прогоняем утилиты проверки:
- Kotlin Style linter check.
- Проверка зависимостей.
- Сборка Docker контейнера.
